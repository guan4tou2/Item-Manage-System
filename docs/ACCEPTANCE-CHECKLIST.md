# Item Manage System Acceptance Checklist

本清單用於逐頁驗收目前專案的主要功能、操作流程與 UI/UX 一致性。

## 1. 登入與帳號

- [ ] 開啟 `/signin` 可正常顯示登入頁。
- [ ] 使用 `admin / admin` 可登入，首次登入會被要求修改密碼。
- [ ] 修改密碼後可正常進入 `/home`。
- [ ] 登出後重新進入受保護頁面會被導回 `/signin`。

## 2. 首頁 `/home`

- [ ] 頁面無水平捲動，桌機與手機都不超出畫面。
- [ ] 搜尋區、統計卡片、物品卡片都可正常顯示。
- [ ] 通知數量 badge 可正常載入，不應因 `429` 導致 UI 壞掉。
- [ ] 物品卡片的收藏、編輯、搜尋相似、QR/條碼按鈕可點擊。
- [ ] FAB 新增按鈕不超出畫面。

## 3. 新增物品 `/additem`

- [ ] 必填欄位提示可見。
- [ ] 自動產生 Item ID 可正常填入。
- [ ] 選照片後可看到預覽。
- [ ] 常用位置快捷按鈕可填入放置地點。
- [ ] 樓層、房間、區域可正常選擇。
- [ ] 缺必填欄位時，表單會阻擋提交並顯示錯誤提示。
- [ ] 成功送出後，物品可在首頁或管理頁找到。

## 4. 編輯物品 `/edititem/<id>`

- [ ] 頁面可顯示目前資料與目前照片。
- [ ] Item ID 為唯讀，不可修改。
- [ ] 常用位置快捷按鈕可快速覆蓋位置。
- [ ] 保固與使用期限狀態可清楚顯示。
- [ ] 儲存變更後資料有正確更新。
- [ ] 刪除 modal 可正常開啟，確認刪除後物品消失。

## 5. 管理物品 `/manageitem`

- [ ] 管理頁可正常載入，不出現 500。
- [ ] 單筆位置快速更新可成功。
- [ ] 批量選取、批量移動、批量刪除可正常操作。
- [ ] 匯出 JSON、CSV、補貨清單按鈕可下載資料。
- [ ] 匯入按鈕會導向新匯入頁 `/import/`。

## 6. 搜尋 `/search`

- [ ] 搜尋頁可顯示摘要條與多條件搜尋表單。
- [ ] 名稱、地點、類型、樓層、房間、區域篩選可組合使用。
- [ ] 清除篩選按鈕可重置條件。
- [ ] 無結果時會顯示空狀態與下一步操作。
- [ ] 搜尋結果卡片不超出畫面。

## 7. 收藏 `/favorites`

- [ ] 收藏頁可正常顯示收藏數量與摘要。
- [ ] 取消收藏後卡片會從頁面移除。
- [ ] 最後一個收藏取消後，會回到空狀態。
- [ ] 空狀態提供返回首頁與前往搜尋的入口。

## 8. 掃描 `/scan`

- [ ] 掃描頁可正常載入。
- [ ] 未授權相機時會顯示權限提示。
- [ ] 相機失敗時會顯示錯誤說明與重試按鈕。
- [ ] 可從掃描頁直接改用手動搜尋。
- [ ] 掃描成功後會顯示成功狀態並導向搜尋結果。
- [ ] 最近掃描歷史可正常顯示。

## 9. 通知摘要 `/notifications/summary`

- [ ] 已過期、即將到期、庫存不足、總警報卡片可正常顯示。
- [ ] 頁首摘要條會顯示到期、更換、庫存與建議順序。
- [ ] 無通知時會顯示空狀態與通知設定入口。
- [ ] 有通知時，已過期、即將到期、更換提醒、低庫存區塊都能正常呈現。

## 10. 通知設定 `/notifications/`

- [ ] 可切換通知啟用狀態。
- [ ] Email、天數、時間、提醒梯度可正常儲存。
- [ ] 儲存後會顯示 inline feedback。
- [ ] 發送測試通知與立即發送通知會顯示成功或失敗訊息。
- [ ] 未啟用通知或未填 Email 時，摘要區文案正確。

## 11. 補貨清單 `/reorder`

- [ ] 頁首摘要條可顯示需補貨、低庫存、建議動作。
- [ ] 需補貨與低庫存列表可正常分開顯示。
- [ ] 快速調整數量按鈕可更新畫面數值。
- [ ] 無補貨項目時，空狀態會提供返回首頁與管理物品入口。

## 12. 統計 `/statistics`

- [ ] 頁首摘要條可顯示資料完整度、主要樓層、主要類型、保養提醒。
- [ ] 類型分佈、樓層分佈、到期狀態、資料完整度、保養狀態分佈區塊可正常顯示。
- [ ] 可從統計頁直接前往通知摘要處理保養提醒。
- [ ] 空資料時不應 500。
- [ ] 頁面不依賴逐項 count 查詢，應能穩定載入。

## 13. 匯入 `/import/`

- [ ] 頁面可正常載入，不出現 500。
- [ ] 可下載 CSV 模板。
- [ ] 上傳 CSV 或 JSON 時會顯示 loading 與結果摘要。
- [ ] 錯誤資料會顯示失敗列與原因。
- [ ] 匯入前可從頁面直接前往備份。
- [ ] `MaintenanceCategory / MaintenanceIntervalDays / LastMaintenanceDate` 可正常匯入。

## 14. 備份 `/backup`

- [ ] 可下載完整備份 JSON。
- [ ] 備份 JSON 需包含正式保養欄位。
- [ ] 選擇還原檔案後會顯示檔名提示。
- [ ] 還原成功後會顯示物品、類型、位置的統計結果。
- [ ] 還原失敗時會顯示錯誤訊息，不應靜默失敗。
- [ ] 備份頁需明確說明：不包含圖片、使用者帳號、通知設定。

## 15. 旅行 `/travel/`

- [ ] 旅行列表頁可正常載入。
- [ ] 重新整理按鈕應留在 `/travel/`，不應誤跳 API endpoint。
- [ ] 建立旅行表單應送到 `/travel/create`。
- [ ] 空狀態可正常顯示。
- [ ] 旅行詳情頁可新增群組、新增物品、更新數量、刪除物品。

## 16. 導覽與共用 UI

- [ ] Navbar、dropdown、tooltip、FAB 在桌機與手機都不超出畫面。
- [ ] `nav-link` hover 不應只剩空白框。
- [ ] tooltip 文字在深色背景下仍可辨識。
- [ ] 管理選單分組清楚，手機展開不錯位。

## 17. 穩定性

- [ ] `/favorites` 不應再出現舊的 JSON 查詢 500。
- [ ] `/api/notifications/count` 應回 `200`，不再因全域 rate limit 頻繁出現 `429`。
- [ ] `/manageitem` 不應再出現 500。
- [ ] 所有主要頁面應無水平捲動與超出瀏覽器畫面的物件。

## 18. 建議驗收順序

1. 先驗登入、首頁、新增、編輯。
2. 再驗搜尋、收藏、管理、掃描。
3. 最後驗通知、補貨、統計、匯入、備份、旅行。
